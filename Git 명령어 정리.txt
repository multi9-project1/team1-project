1. 최신 내용 받기
git pull origin main

2. 작업한 파일 전부 스테이징 
git add .

3. 커밋하기 (라벨 없어도 됨)
git commit -m "라벨: 작업 내용"

4. git에 올리기
git push origin main



- 현재 상태 확인 (수시로)
git status

- 특정 파일만 스테이징
git add 파일명

- 커밋 목록 확인
git log --oneline


- 커밋 메세지 변경
git commit --amend -m "새 커밋 메시지"

- 커밋 취소 (add 남아있음)
git reset --soft HEAD~1

- 커밋/add 취소
git reset HEAD~1



커밋 라벨 (없어도 괜찮음)

feat
새 기능 추가
예: 검증셋 분리 기능 추가, 새로운 모델 코드 추가

fix
버그 수정
예: 에러 나는 코드 고침, 잘못된 전처리 수정

refactor
기능은 같지만 코드 구조를 개선
예: 전처리 흐름을 더 깔끔하게 바꿈, 누수 없게 순서 수정

chore
잡일성 변경, 기타 작업
예: 주석 수정, 파일 정리, 설정 변경, 불필요한 파일 삭제

docs
문서 수정
예: README, 노션용 md, 설명 파일 수정

style
코드 스타일만 수정
예: 들여쓰기, 공백, 줄바꿈, 포맷팅
기능 변화 없음

test
테스트 코드 추가/수정
예: 테스트 파일 작성, 테스트 케이스 추가
